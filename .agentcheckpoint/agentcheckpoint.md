# Agent Checkpoint — selimsolutions

> Compact orientation doc for AI agents. Read this first.

## Intent

A **fresh, mostly-scaffolded full-stack TypeScript monorepo** generated by
[Better-T-Stack](https://better-t-stack.dev) (v3.33.1) on 2026-06-19. There is
no business logic yet — it's a clean starting point with auth, DB, and a
type-safe API wired end-to-end, ready to build features on.

The only real "feature" so far: a home page that pings the API health check and
shows Connected/Disconnected, plus working email/password authentication.

## Stack

| Layer       | Choice                                                      |
|-------------|------------------------------------------------------------|
| Monorepo    | Turborepo + npm workspaces (`apps/*`, `packages/*`)        |
| Frontend    | React 19, TanStack Router (file-based), TailwindCSS v4, Vite 8 |
| Backend     | Express 5 on Node.js                                        |
| API         | oRPC (end-to-end type-safe RPC + OpenAPI reference)         |
| Auth        | Better-Auth (email/password), cookie sessions              |
| DB / ORM    | MySQL (Docker) + Prisma 7 via `@prisma/adapter-mariadb`    |
| UI          | shadcn/ui primitives shared in `packages/ui`               |
| Env         | `@t3-oss/env-core` + Zod validation                        |
| Tooling     | Biome (lint/format), tsx, tsdown                            |

## Layout

```
apps/
  web/      React + TanStack Router frontend (port 5173)
  server/   Express + oRPC backend (port 3000)
packages/
  api/      oRPC routers, procedures, context (the API contract)
  auth/     Better-Auth config (createAuth)
  db/       Prisma client factory + schema + docker-compose
  ui/       Shared shadcn/ui components & styles
  env/      Zod-validated env (server.ts / web.ts)
  config/   Shared tsconfig bases
```

## How it fits together

- **Server** (`apps/server/src/index.ts`): Express mounts Better-Auth at
  `/api/auth/*`, oRPC RPC handler at `/rpc`, and OpenAPI reference at
  `/api-reference`. CORS is locked to `CORS_ORIGIN` with credentials.
- **API contract** (`packages/api`): `appRouter` currently has two procedures —
  `healthCheck` (public, returns "OK") and `privateData` (protected, returns the
  session user). `protectedProcedure` enforces auth via middleware that throws
  `UNAUTHORIZED` when there's no session. Context is built per-request from the
  Better-Auth session.
- **Web** (`apps/web`): Creates a typed oRPC client (`AppRouterClient`) over
  `RPCLink` to `${VITE_SERVER_URL}/rpc`, wrapped in TanStack Query. Auth via
  `better-auth/react` client. Routes: `/` (home/health) and `/login`.
- **DB** (`packages/db`): `createPrismaClient()` parses `DATABASE_URL` into a
  MariaDB adapter connection. Schema is Better-Auth's: `User`, `Session`,
  `Account`, `Verification`. Generated client lives in `prisma/generated/`.

## Run it

```bash
npm install
npm run db:start    # docker compose up MySQL (db: selimsolutions, user/password)
npm run db:push     # apply schema
npm run dev         # all apps (web :5173, server :3000)
```

Other: `dev:web`, `dev:server`, `db:studio`, `db:migrate`, `db:generate`,
`check` (Biome), `check-types`.

## Env required

- `apps/server/.env`: `DATABASE_URL`, `BETTER_AUTH_SECRET` (≥32 chars),
  `BETTER_AUTH_URL`, `CORS_ORIGIN`, `NODE_ENV`
- `apps/web/.env`: `VITE_SERVER_URL`
- `.env.example` files exist in both `apps/web/` and `apps/server/` as templates

## Databases (dev vs production)

Two environments, selected by the env file that's loaded:

| Env  | File                          | DATABASE_URL target                       |
|------|-------------------------------|-------------------------------------------|
| dev  | `apps/server/.env`            | local Docker MySQL (`localhost:3306`)     |
| prod | `apps/server/.env.production` | Hostinger MySQL (`srv838.hstgr.io:3306`, db `u657067197_selim`) |

- Both env files are **gitignored** (secrets never committed).
- `packages/db/prisma.config.ts` picks the env file by `NODE_ENV`
  (`production` → `.env.production`, else `.env`).
- **Password URL-encoding**: the prod password contains `@` and `#`, encoded in
  the URL as `%40`/`%23`. `packages/db/src/index.ts` `decodeURIComponent`s the
  user/password before handing them to the MariaDB driver — required for special
  chars to work.
- Prod schema is **already pushed** (auth tables: user/session/account/verification).

### Prod DB commands (NODE_ENV=production via cross-env)

```bash
npm run db:push:prod      # sync schema to production (no migration history)
npm run db:migrate:prod   # apply committed migrations (prisma migrate deploy)
npm run db:studio:prod    # open Prisma Studio against production
```

⚠️ Before deploying, set real values in `apps/server/.env.production` for
`BETTER_AUTH_URL` and `CORS_ORIGIN` (currently placeholders) and generate a
production-only `BETTER_AUTH_SECRET`.

## Notes for agents

- **To add an API endpoint**: add a procedure to `appRouter` in
  `packages/api/src/routers/index.ts` — types flow to the web client automatically.
- **To add DB models**: edit `packages/db/prisma/schema/*.prisma`, then `db:push`/`db:generate`.
- **To add shared UI**: `npx shadcn@latest add <comp> -c packages/ui`.
- Git: single `initial commit` on `master`; main branch is `main`.
- `.agents/skills/` contains bundled reference skills (turborepo, prisma,
  better-auth, shadcn, web-design) — useful background docs.
- `bts.jsonc` records the exact generator config and is safe to delete.
- Biome VCS integration is enabled (`useIgnoreFile: true`) — respects `.gitignore`.
- Vite dev server runs on port **5173** (matches `CORS_ORIGIN` in server `.env`).
- `apps/server/.env.production` exists for production commands (`db:push:prod`, etc.).
